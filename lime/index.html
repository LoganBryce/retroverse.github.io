What is Lime?

Lime is a game engine made with the LÃ–VE2D framework. It is designed to be fast, flexible and easy to use. It is built using a superset of ECS called Jungle and a superset of MoonScript called MoonShine.

Why use Lime?

It is easy to setup thanks to its template system and it includes everything you need to get started without wasting time setting up libraries. It also includes a language designed with speed in mind that helps you turn your ideas into code with less hassle.

What's MoonShine?

MoonScript is superset of Lua and MoonShine in a superset of MoonScript that aims to add functionality and make the language more familiar for people coming from other languages.

What's a Jungle?

Jungle is a similiar system to an Entity Component System However it has some slight differences, the elements of a jungle are below.

  • Species

    Species are like classes of components. They have behaviours and are instantiated to create entities. Most entities are created by a helper function created by the user. An example species is a player.

  • Behaviours

    Behaviours are functionality and are found on species. They have events which run on entities at certain times, namely Spawn, Update & Draw. An example behaviour is movement.

  • Entities

    Entities Are runtime instances of species. For example, if a pig is a species, then a single pig in your game is an entity.

  • Operations

    Operations are methods which are run on all entities that have a particular behaviour. An example operation is a draw call where you want to organise sprites based on their depth.

What Species are Included?

Some basic behaviours that are included and are also easily modifiable, many implement the engines libraries.

  • movement

    Allows you to control an entity with keyboard, joysticks or controllers.

  • physics

    Simple drop physics, no gravity implemented but easily extensible.

  • physics-anim

    Update sprite based on last input and speed.

  • sprite

    Render an image at a sprites position, required for enitity to have an associated position.

  • sprite-states

    Allows multiple sprites to be easily switched between. i.e run or idle.
